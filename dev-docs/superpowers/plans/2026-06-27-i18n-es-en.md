# Selector de idioma (español / inglés) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Añadir a Oraltags un selector de idioma en la cabecera (español por defecto + inglés) que traduce el núcleo de la interfaz en caliente, sin tocar el esquema de anotación, y recuerda la elección entre sesiones.

**Architecture:** Un único `i18n/translation.json` (es→en) alimenta dos mecanismos que comparten claves: `shiny.i18n` (`usei18n` + `i18n$t()` en la UI estática, conmutado en caliente con `update_lang`) para el texto de la UI, y un helper puro `tr(key, lang)` para las notificaciones generadas en servidor (que `update_lang` no cubre por sesión). El idioma se persiste en `config/preferencias.txt`.

**Tech Stack:** R 4.5+, Shiny, shiny.i18n 0.3.0, base R + `jsonlite`. `shiny.i18n` pasa a dependencia obligatoria.

## Global Constraints

- App de un solo archivo `etiquetador_oral.R`; NO sourcear el archivo entero en tests.
- Idiomas: **es (por defecto) + en**. Solo se traduce el **núcleo** (pestañas/subpestañas, botones principales, cabeceras de sección, etiquetas de controles, subtítulo, y las notificaciones listadas). NO se traduce el esquema de anotación (`anot_defs`: 33 etiquetas + 181 categorías) ni el texto dinámico de servidor (títulos de `plot`, informe de Métricas, encabezados de DT).
- `shiny.i18n` 0.3.0 API: `Translator$new(translation_json_path=...)`, `i18n$set_translation_language("es")`, `i18n$t("…")`, `usei18n(i18n)`, `update_lang(language, session)` (idioma primero).
- Mecanismo doble: **UI estática** se envuelve con `i18n$t("clave_es")`; **notificaciones** (server) con `tr("clave_es", session_lang())`. Ambos usan las mismas claves del `translation.json`. La clave es SIEMPRE la cadena en español.
- Persistencia: clave `idioma` en `config/preferencias.txt` (default "es"); el selector la guarda; el arranque la aplica por sesión con `update_lang` + `updateSelectInput`.
- El traductor global arranca en "es" (la UI se construye en español); el cambio es por sesión.
- Test de consistencia (forward): toda clave usada en `i18n$t("…")`/`tr("…", …)` debe existir con `en` no vacío en el JSON. Completitud (toda clave del JSON está envuelta) se comprueba al final.
- Verificación UI: `Rscript -e 'invisible(parse("etiquetador_oral.R")); cat("PARSE OK\n")'` y `Rscript -e 'shinyApp <- function(ui, server) cat("BUILT OK\n"); suppressWarnings(source("etiquetador_oral.R"))'`. Tests: `Rscript -e 'testthat::test_dir("tests/testthat", stop_on_failure = TRUE)'`.
- Hay archivos de datos rastreados modificados (`analisis/analisis_todos.txt`, `jueces/.DS_Store`) y carpetas sin rastrear: usar SIEMPRE `git add` con rutas concretas; nunca `git add .`/`git add -A`.

```r
# R/i18n_util.R — utilidades de i18n (puras, base R + jsonlite)

load_i18n_dict <- function(json_path) {
  empty <- list(es = character(0), en = character(0))
  if (!file.exists(json_path)) return(empty)
  j <- tryCatch(jsonlite::fromJSON(json_path, simplifyVector = TRUE),
                error = function(e) NULL)
  if (is.null(j) || is.null(j$translation)) return(empty)
  tr_df <- j$translation
  list(es = as.character(tr_df$es), en = as.character(tr_df$en))
}

tr <- function(key, lang, dict) {
  if (is.null(lang) || lang != "en") return(key)
  i <- match(key, dict$es)
  if (is.na(i)) return(key)
  en <- dict$en[i]
  if (is.na(en) || !nzchar(en)) key else en
}

i18n_used_keys <- function(code_path) {
  code <- paste(readLines(code_path, warn = FALSE, encoding = "UTF-8"), collapse = "\n")
  pat <- '(?:i18n\\$t|\\btr)\\(\\s*"((?:[^"\\\\]|\\\\.)*)"'  # \btr evita casar attr(, substr(…
  m <- gregexpr(pat, code, perl = TRUE)
  keys <- regmatches(code, m)[[1]]
  if (length(keys) == 0) return(character(0))
  vals <- sub(pat, "\\1", keys, perl = TRUE)
  vals <- gsub('\\\\"', '"', vals)   # des-escapar comillas
  unique(vals)
}

i18n_missing_translations <- function(code_path, json_path) {
  used <- i18n_used_keys(code_path)
  dict <- load_i18n_dict(json_path)
  has_en <- function(k) {
    i <- match(k, dict$es)
    !is.na(i) && !is.na(dict$en[i]) && nzchar(dict$en[i])
  }
  used[!vapply(used, has_en, logical(1))]
}
```
